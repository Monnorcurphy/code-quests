## Informational Notes

### 1. PNG files are synthetic stubs

`assets/CREDITS.md` discloses (correctly) that the PNG files currently in the tree are generated by `scripts/gen-asset-stubs.mjs` and not the actual Kenney/0x72 packs they are attributed to. The spec accepts this implicitly via the testable criterion "every PNG ≥ 1KB" — and the stubs do satisfy that. Downstream tasks (`elephant`, `fox`) that actually consume the art will visually surface noise instead of pixel-art tiles and must swap in the real downloads before phase exit. This is acknowledged in CREDITS.md ("Replace them with the real packs downloaded from the sources listed above when preparing a release build") and progress.md. Not filed as a bug because the spec criterion is met as written, but flag for the elephant/fox builders.

### 2. `check:assets` not wired into an aggregate verify script

Spec line 16 says "include in the verify chain". The script `pnpm check:assets` exists and works, but the repo has no aggregate verify script (only `build`, `test`, `lint`, `typecheck`, `dev`). The factory's ralph.sh / verify.sh apparently invoke individual commands. The `check:assets` invocation should be added to whatever check runner orchestrates pre-PR verification (likely `scripts/verify.sh` per project layout — outside the scope of this task, but worth flagging for the next task that touches the verify chain). Not filed as a bug because there is no defined "verify chain" target to wire into.

### 3. CC-BY license version drift between CREDITS.md and manifest.json

`CREDITS.md` records the 0x72 license as **"CC-BY 4.0"**; `manifest.json` records it as **"CC-BY"** (no version). Both are recognizable, but using the same string in both files would reduce ambiguity for the CI check and future maintainers. Cosmetic only; not filed.

### 4. `asset-loader.test.ts` reads files via Node `fs` rather than via a Phaser mock

The test is effectively an integration check against the symlinked `public/assets/` tree. This is good — it would catch a broken symlink or a missing file at unit-test time, before the dev server is even started. No issue, just noting the deliberate design.

## Final Verdict

**FAIL** — 2 bug files filed (1 HIGH, 1 LOW).

All automated gates pass. The HIGH bug is a missed spec requirement (Kenney Tiny Dungeon pack), not a runtime defect. The LOW bug is a code-style fix in the asset license check script.
